## 🎯 `RandomCandidateSelector`

The `RandomCandidateSelector` is a utility class responsible for selecting a valid candidate from an RDF graph at random and marking it as selected. It is designed to work with graphs structured according to the `slowmo` ontology.

---

### 📌 Purpose

This component encapsulates the logic for:

* Identifying valid candidates in an RDF graph
* Applying selection criteria
* Randomly choosing one candidate
* Annotating the graph with the selection result

It is particularly useful in workflows where multiple candidate entities exist and one must be chosen non-deterministically based on defined constraints.

---

### 🧠 Selection Logic

A resource is considered a **valid candidate** if it satisfies both of the following:

1. It has type:

   ```
   rdf:type slowmo:Candidate
   ```

2. It has at least one:

   ```
   slowmo:AcceptableBy <some agent>
   ```

Only candidates meeting these criteria are eligible for selection.

---

### ⚙️ Method: `select_candidate`

```python
select_candidate(performer_graph: Graph) -> Resource
```

#### Parameters

* `performer_graph` (`rdflib.Graph`):
  The RDF graph containing candidate resources and their relationships.

#### Returns

* `rdflib.resource.Resource`:
  The selected candidate.

#### Behavior

* Filters all valid candidates from the graph
* Randomly selects one candidate using Python’s `random.choice`
* Adds the following triple to the graph:

  ```
  <selected_candidate> slowmo:Selected true
  ```
* Returns the selected candidate as a `Resource` object

---

### 🔁 Side Effects

This method **modifies the input graph in place** by adding a triple indicating which candidate was selected. Consumers of this method should be aware that the graph is mutated.

---

### ⚠️ Error Handling

* If no valid candidates are found, the method raises an:

  ```

  ```

  (due to `random.choice` being called on an empty list)
